# Data: Synthea synthetic patient data

This project trains on **synthetic clinical data generated by Synthea**, an open-source patient generator from The MITRE Corporation. No real patient data is used. The system is designed to swap in real EHR data (e.g., MIMIC-IV) by replacing the ingestion layer.

## Why synthetic data?

- **No PHI**: the public repository, notebooks, and blog post can include sample data and outputs without risk.
- **Fast iteration**: generate 1,000 or 100,000 patients in minutes.
- **Realistic distributions**: Synthea models common diseases with epidemiologically grounded prevalence, demographics, and care pathways.
- **Codes included**: every condition, procedure, and medication is tagged with ICD-10, SNOMED, RxNorm, or LOINC codes.

## What Synthea provides — and what we add

Synthea produces *structured* encounter data (conditions, procedures, medications, vitals, etc.). It does **not** produce free-text clinical notes by default.

For this project we need free-text notes as model input. We synthesize them from Synthea's structured data using an LLM. This is a deliberate design choice: it gives us a controlled pipeline where ground-truth codes are known (because Synthea generated them) and the notes can vary in style and detail.
